# Лабораторная работа №3. Клеточные автоматы. Лесные пожары

## Описание модели

### Сетка и состояния клеток

Модель представляет собой двумерный клеточный автомат, реализованный на прямоугольной сетке размером `150 × 150`. Для максимальной производительности вычисления векторизованы с использованием матричных операций библиотеки `NumPy`.

Каждая клетка находится в одном из пяти состояний (кодируются целыми числами для быстрой обработки):

```python
EMPTY = 0    # Земля (Темно-коричневая)
TREE = 1     # Дерево (Зеленая)
BURNING = 2  # Огонь (Желтая)
ASH = 3      # Пепел (Темно-серая)
OBSTACLE = 4 # Преграда (Синяя)
```

У каждой клетки также есть скрытый параметр — индивидуальный таймер горения, хранящийся в отдельной матрице `self.burn_time`.

### Векторизованный расчет соседей

Вместо медленного перебора каждой клетки в цикле `for`, в модели применяется сдвиг матриц (`np.roll`). Для базового распространения огня используется **окрестность фон Неймана** (4 соседа: Север, Юг, Запад, Восток). 

Чтобы огонь не «телепортировался» через края карты (особенность функции `np.roll`), граничные значения принудительно обнуляются:

```python
is_burn_int = is_burning.astype(float)

# Соседи на расстоянии 1 клетки (базовое давление огня)
N = np.roll(is_burn_int, 1, axis=0); N[0, :] = 0
S = np.roll(is_burn_int, -1, axis=0); S[-1, :] = 0
W = np.roll(is_burn_int, 1, axis=1); W[:, 0] = 0
E = np.roll(is_burn_int, -1, axis=1); E[:, -1] = 0
```

### Шаг симуляции

На каждом шаге симуляции состояния пересчитываются синхронно для всей сетки. Создаются копии массивов (`new_grid`, `new_burn_time`), к ним применяются правила через логические маски NumPy, после чего новые матрицы заменяют старые. Случайности обрабатываются единой матрицей `rand_mat = np.random.rand(...)`.

## Основные правила

### Правило 1. Горящая клетка выгорает

При возгорании клетка получает случайный таймер жизни огня от 3 до 6 шагов. На каждом тике таймер уменьшается. Когда он доходит до нуля, огонь тухнет и клетка переходит в выгоревшее состояние (`ASH`):

```python
new_burn_time[is_burning] -= 1
just_finished = is_burning & (new_burn_time <= 0)
new_grid[just_finished] = ASH
```

### Правило 2. Распространение огня

Вероятность поджога клетки зависит от «давления огня» со стороны соседей. Каждый горящий сосед вплотную добавляет 15% к шансу возгорания. 

```python
fire_pressure = (N + S + W + E) * 0.15
total_ignition_prob = fire_pressure + spark_pressure + f_lightning

ignited = (self.grid == TREE) & (rand_mat < total_ignition_prob)
new_grid[ignited] = BURNING
```

### Правило 3. Спонтанное возгорание (молния)

С вероятностью `f` (регулируется ползунком) случайное дерево может загореться само по себе, имитируя удар молнии. Это значение (`f_lightning`) просто прибавляется к общему шансу возгорания клетки.

### Правило 4. Восстановление леса

Пустая земля зарастает новыми деревьями с вероятностью `p` (задается ползунком), замыкая базовый цикл модели:

```python
grow_empty = (self.grid == EMPTY) & (rand_mat < p_growth)
new_grid[grow_empty] = TREE
```

## Дополнительные правила (Особенности реализации)

### Доп. правило 1. Механика пепла и эффект удобрения

В отличие от классической модели, выгоревшая клетка не становится пустой землей мгновенно, а превращается в пепел (`ASH`). 
Во-первых, пепел со временем осыпается и превращается в пустую землю с вероятностью 2% за шаг:
```python
ash_decay = (self.grid == ASH) & (rand_mat < 0.02)
new_grid[ash_decay] = EMPTY
```
Во-вторых, реализован **эффект удобрения**: деревья растут на выгоревшем пепле в 5 раз быстрее, чем на обычной земле. Это создает реалистичный цикл обновления экосистемы после лесного пожара:
```python
grow_ash = (self.grid == ASH) & (rand_mat < p_growth * 5.0)
new_grid[grow_ash] = TREE
```

### Доп. правило 2. Искры (Огонь на расстоянии)

Огонь способен перекидываться через пустые клетки (или тонкие преграды). Для этого рассчитывается дополнительное давление от «искр» — соседей на расстоянии в 2 клетки от текущей. Искры добавляют 1% к шансу возгорания:

```python
        N = np.roll(is_burn_int, 1, axis=0)
        N[0, :] = 0
        S = np.roll(is_burn_int, -1, axis=0)
        S[-1, :] = 0
        W = np.roll(is_burn_int, 1, axis=1)
        W[:, 0] = 0
        E = np.roll(is_burn_int, -1, axis=1)
        E[:, -1] = 0


        N2 = np.roll(is_burn_int, 2, axis=0)
        N2[0:2, :] = 0
        S2 = np.roll(is_burn_int, -2, axis=0)
        S2[-2:, :] = 0
        W2 = np.roll(is_burn_int, 2, axis=1)
        W2[:, 0:2] = 0
        E2 = np.roll(is_burn_int, -2, axis=1)
        E2[:, -2:] = 0


        fire_pressure = (N + S + W + E) * 0.15
        spark_pressure = (N2 + S2 + W2 + E2) * 0.01
```

### Доп. правило 3. Интерактивная Роза Ветров

Направление и сила ветра задаются через UI (9 кнопок направлений и ползунок силы). Ветер смещает пламя, добавляя модификатор к давлению огня с подветренной стороны. Например, если ветер дует на Восток (`wind_dx > 0`), огонь активнее перекидывается с Запада (`W` и `W2`):

```python
if self.wind_dx > 0: # Ветер вправо
    fire_pressure += W * wind_str
    spark_pressure += W2 * wind_str * 0.5
```
Это приводит к образованию характерных вытянутых фронтов пожара, движущихся по направлению ветра.

### Доп. правило 4. Процедурная генерация ландшафта и непроходимые преграды

При старте симуляции мир не генерируется случайным шумом, а проходит через алгоритм процедурной генерации на основе клеточного автомата (схожего с игрой «Жизнь»). 
Изначально раскидывается случайный шум преград, который затем «сглаживается» за 4 итерации (если вокруг много преград — клетка тоже становится преградой, образуя сплошные массивы). Эти зоны выступают в роли состояния `OBSTACLE`, которое абсолютно игнорируется огнем и выступает естественным барьером, блокирующим распространение пламени:

```python
new_grid[self.grid == OBSTACLE] = OBSTACLE
```
