# Лабораторная работа №8. Пуассоновский поток заявок на сервер

В данной лабораторной работе реализовано имитационное моделирование простейшего потока событий на примере поступления заявок на сервер. Программа позволяет задавать интенсивность потока, длину наблюдаемого интервала и число повторных прогонов, после чего сравнивает полученное распределение с теоретическим законом Пуассона.

---

## 1. Постановка задачи

Необходимо смоделировать поток заявок, поступающих на сервер, и выполнить статистический анализ числа событий за фиксированный интервал времени.

**Событие:** поступление одной заявки на сервер.

**Входные параметры:**
*   интенсивность потока $\lambda$ — среднее число заявок в секунду;
*   интервал наблюдения $T$;
*   число независимых прогонов $N$.

**Требуется:**
1. Определить число заявок на каждом интервале длины $T$.
2. Построить эмпирическое распределение числа заявок.
3. Вычислить эмпирическое среднее и дисперсию.
4. Сравнить результаты с теоретическим распределением Пуассона.
5. Сделать вывод о согласии модели с теорией.

---

## 2. Математическая модель

Для простейшего потока событий предполагается, что промежутки времени между соседними заявками независимы и имеют экспоненциальное распределение с параметром $\lambda$.

Если $X$ — число заявок за интервал времени $T$, то:

$$
X \sim Pois(\mu), \qquad \mu = \lambda T
$$

Вероятность получить ровно $k$ заявок вычисляется по формуле:

$$
P(X = k) = \frac{\mu^k e^{-\mu}}{k!}
$$

Для распределения Пуассона математическое ожидание и дисперсия совпадают:

$$
M[X] = D[X] = \mu
$$

---

## 3. Алгоритм моделирования

### 3.1. Генерация потока заявок

Один прогон моделирования выполняется последовательным накоплением случайных интервалов между поступлениями заявок:

```python
t += -np.log(np.random.rand()) / lmbda
```

Если очередной момент времени не превышает $T$, заявка учитывается. Когда момент времени становится больше $T$, прогон завершается.

### 3.2. Серия экспериментов

Для статистической устойчивости один прогон повторяется $N$ раз:

```python
all_counts = [len(simulate_poisson_stream(lmbda, T)) for _ in range(N)]
```

В массиве `all_counts` хранится число заявок, полученное в каждом независимом интервале наблюдения.

### 3.3. Статистическая обработка

По результатам серии прогонов вычисляются:
*   эмпирическое среднее;
*   выборочная дисперсия;
*   частоты появления каждого числа заявок.

Частоты нормируются на $N$ и отображаются на графике как эмпирические вероятности. Теоретическая кривая строится по формуле распределения Пуассона.

## 4. Реализация программы

Приложение написано в одном файле `lab8.py`. В него перенесена логика исходного модуля `core.py` и реализован графический интерфейс на **PyQt6**.

### Основные элементы интерфейса

*   Поля ввода параметров $\lambda$, $T$ и $N$.
*   Кнопка запуска симуляции.
*   График сравнения эмпирического и теоретического распределений.
*   Карточки со значениями:
    *   $\mu = \lambda T$;
    *   эмпирическое среднее;
    *   эмпирическая дисперсия;

*   Автоматический текстовый вывод о согласии результатов с теорией.

Интерфейс повторяет структуру исходной версии: сверху располагаются параметры, в центре — график распределения, справа — статистические показатели, снизу — итоговый вывод.

## 5. Пример интерпретации результатов

Для параметров по умолчанию:

*   $\lambda = 5$ заявок/с;
*   $T = 3$ с;
*   $N = 1000$ прогонов.

Теоретическое значение:

$$
\mu = \lambda T = 5 \cdot 3 = 15
$$

Следовательно, при большом числе прогонов эмпирическое среднее и дисперсия должны быть близки к 15. Небольшие отклонения являются нормальными, так как моделирование использует случайные числа.
