# Лабораторная работа №5 — Моделирование случайных событий

## Описание задачи
**Цель работы** — реализовать программное моделирование случайных событий с использованием генератора псевдослучайных чисел на основе заданных законов распределения. 

В рамках работы разработаны две независимые математические модели, объединенные в графическом интерфейсе (GUI):
1. **Оракул** — генератор бинарных ответов «Да / Нет» с динамически изменяемой вероятностью.
2. **Игровой автомат** — моделирование системы дискретных случайных величин с использованием метода кумулятивных вероятностей (взвешенных шансов).

---

## Часть 1. Приложение «Оракул» (Да / Нет)

### Идея
Модель представляет собой бинарный случайный эксперимент. Каждый вызов независимо возвращает один из двух возможных исходов — «ДА» или «НЕТ». Вероятность выпадения положительного ответа ($p$) задается пользователем.

### Реализация
```python
import random

def get_yes_no(p=0.5):
    return "ДА" if random.random() < p else "НЕТ"
```

**Принцип работы:**
Функция `random.random()` возвращает равномерно распределённое вещественное число $\alpha$ из интервала $[0, 1)$. Если сгенерированное число $\alpha < p$, результатом становится «ДА», иначе — «НЕТ». При установке параметра $p = 0.8$, положительный исход будет выпадать в среднем в 80% случаев.

## Часть 2. Приложение «Игровой автомат»

### Идея
Автомат состоит из трех независимых барабанов, на каждом из которых расположено 6 символов. Реализована система **взвешенных вероятностей**: пользователь задает вес ($w_i$) каждого символа, что позволяет делать определенные исходы редкими, а другие — частыми.

### Реализация — Метод обратного преобразования

```python
SYMBOLS = ["Вишня", "Лимон", "Апельсин", "Виноград", "Алмаз", "Семерка"]

def get_weighted_symbol(symbols, weights):
    m = len(symbols)
    total_weight = sum(weights)
    
    alpha = random.random()
    cumulative_p = 0.0
    
    for k in range(m):
        p_i = weights[k] / total_weight  
        cumulative_p += p_i
        
        if alpha < cumulative_p:
            return symbols[k]
            
    return symbols[-1]

def get_casino_result(symbols, weights):
    return [get_weighted_symbol(symbols, weights) for _ in range(3)]
```

### Математическое обоснование
Алгоритм опирается на вычисление кумулятивной функции распределения $F(k)$:

1. Вычисляется сумма всех заданных весов: $W = \sum_{i=0}^{m-1} w_i$.
2. Вычисляется вероятность $p_i$ для каждого символа: $p_i = \frac{w_i}{W}$.
3. Генерируется псевдослучайное число $\alpha \sim U(0, 1)$.
4. Цикл последовательно накапливает вероятности: $F(k) = \sum_{j=0}^{k} p_j$.
5. Возвращается первый символ, для которого выполняется условие $\alpha < F(k)$.

Геометрически это означает разбиение единичного отрезка $[0, 1]$ на $m$ частей, длина каждой из которых пропорциональна заданному весу символа. В какой отрезок попадет точка $\alpha$, тот символ и выбирается. Процедура повторяется независимо 3 раза для симуляции спина.
