# Лабораторная работа: Марковская модель погоды (Непрерывное время - CTMC)

В данной лабораторной работе реализовано имитационное моделирование смены погодных условий (Ясно, Облачно, Пасмурно) с использованием **цепей Маркова с непрерывным временем (CTMC)**.

---

### **1. Постановка задачи**
Целью работы является моделирование динамической системы с тремя состояниями:
*   **1 — Ясно ☀️**
*   **2 — Облачно ⛅**
*   **3 — Пасмурно 🌧**

В отличие от дискретных моделей, время в данной системе течет **непрерывно**. Переход из одного состояния в другое может произойти в любой момент. В интерфейсе программы симуляция визуализируется пошагово (по "дням" для удобства восприятия), однако внутри алгоритма расчет нахождения в каждом состоянии ведется с точностью до долей единицы времени на основе экспоненциального распределения.

---

### **2. Математическая модель**
В качестве основы модели используется **матрица интенсивностей переходов (Q-матрица)**. 
* Элемент $q_{ij}$ (при $i \neq j$) задает интенсивность (скорость) перехода из состояния $i$ в состояние $j$.
* Диагональные элементы $q_{ii}$ строго равны сумме остальных элементов строки с обратным знаком: $q_{ii} = -\sum_{j \neq i} q_{ij}$. Они отражают общую интенсивность выхода системы из состояния $i$.

**Использованная в симуляции Q-матрица (по умолчанию):**

|              | Ясно ($j=1$) | Облачно ($j=2$) | Пасмурно ($j=3$) |
|:-------------|:----:|:-------:|:--------:|
| **Ясно ($i=1$)**     | **-0.80** |  0.60   |   0.20   |
| **Облачно ($i=2$)**  | 0.40 |  **-0.70**   |   0.30   |
| **Пасмурно ($i=3$)** | 0.10 |  0.50   |   **-0.60**   |

---

### **3. Алгоритм моделирования**
1.  **Генерация времени в текущем состоянии**: Если система находится в состоянии $i$, время её пребывания в нём является случайной величиной, распределенной по экспоненциальному закону с параметром $\lambda_i = -q_{ii}$. Программа генерирует этот интервал времени $\Delta t$ функцией `random.expovariate(rate_out)`.
2.  **Выбор следующего состояния**: Когда время $\Delta t$ истекает, происходит смена погоды. Вероятность перехода в конкретное состояние $j$ вычисляется пропорционально интенсивностям: $p_{ij} = \frac{q_{ij}}{\lambda_i}$. Выбор реализуется методом рулетки (в функции `get_next_state`).
3.  **Теоретическое (стационарное) распределение ($\pi$)**: Определяет долю времени, которую система проведет в каждом состоянии на бесконечности. Находится путем решения системы линейных уравнений $\pi Q = 0$ при условии нормировки $\sum \pi_i = 1$.
4.  **Статистическая обработка**: Программа суммирует **точное непрерывное время** (`exact_time_in_states`), проведенное в каждом состоянии. Эмпирическая доля вычисляется как отношение времени в конкретном состоянии к общему времени симуляции.

---

### **5. Анализ и выводы**
1.  **Адекватность непрерывной модели**: Реализованный алгоритм корректно обрабатывает экспоненциальное распределение времени пребывания в состояниях. Эмпирические доли времени с высокой точностью (погрешность менее $0.005$) совпали с теоретическим распределением, найденным через матрицу интенсивностей $Q$.
2.  **Анализ системы**: Система проводит подавляющее большинство времени (около 43.8%) в состоянии "Облачно". Это математически обосновано Q-матрицей: состояние "Облачно" имеет сильный приток из других состояний ($q_{12} = 0.6$, $q_{32} = 0.5$) и сравнительно небольшую интенсивность выхода ($-q_{22} = 0.7$, что меньше, чем у "Ясно"). Состояние "Ясно", напротив, система покидает быстрее всего (интенсивность выхода $0.8$).
3.  **Особенности реализации**: Пошаговое визуальное отображение (по дням) скрывает внутри себя более точный физический процесс. В рамках одного "визуального" дня погода может поменяться несколько раз, либо не меняться несколько дней подряд — программа точно учитывает эти моменты времени, прибавляя к статистике соответствующие `dt`, что делает статистику высокоточной.
